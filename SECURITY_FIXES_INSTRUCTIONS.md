# تعليمات إصلاح أخطاء Supabase (مشروع Firebase Auth)

بما أنك تستخدم **Firebase Auth** لتسجيل الدخول، فإن تعاملك مع تحذيرات Supabase ينقسم لقسمين:

## 1. تحذيرات يمكنك تجاهلها (تخص Supabase Auth فقط)
التحذير التالي يظهر لأن Supabase يفترض أنك تستخدمه لإدارة المستخدمين، وبما أنك تستخدم Firebase فلا داعي للقلق بشأنه:
*   🔽 **تجاهل التحذير:** `Supabase Auth prevents the use of compromised passwords...`
    *   **السبب:** أنت تدير كلمات المرور والحماية من طرف Google/Firebase، لذا إعدادات "HaveIBeenPwned" في Supabase لن تؤثر عليك.

## 2. تحذيرات يجب إصلاحها (تخص أمان وكفاءة قاعدة البيانات)
التحذيرات التالية تخص **البنية الداخلية لقاعدة البيانات** (Postgres) والدوال (Functions) التي قمنا بكتابتها. هذه مهمة جداً سواء كنت تستخدم Firebase أو غيره، لأنها تحمي البيانات من التلاعب عبر ثغرات الـ SQL.

*   ✅ **يجب إصلاح:** `Function ... has a role mutable search_path`
    *   *الخطر:* يمكن للمهاجمين استغلال الدوال التي ليس لها مسار بحث ثابت لتفيذ أوامر خبيثة.
*   ✅ **يجب إصلاح:** `Extension pg_trgm is installed in the public schema`
    *   *السبب:* تنظيمي وأمني؛ يفضل عزل الإضافات البرمجية عن جداول البيانات.

### خطوات الحل:
لقد قمت بتجهيز كود SQL يقوم بإصلاح هذه المشاكل الفنية أوتوماتيكياً.

1.  افتح **Supabase Dashboard**.
2.  انتقل إلى **SQL Editor** (أيقونة الورقة والقلم في القائمة الجانبية).
3.  اضغط **New Query**.
4.  انسخ الكود الموجود في الملف: `supabase/security_fixes.sql` (الموجود في ملفات مشروعك).
5.  الصقه في المحرر واضغط **Run**.

بعد التشغيل، ستختفي التحذيرات المتعلقة بـ Security و Performance الخاصة بالدالة `search_path` وإضافة `pg_trgm`.
